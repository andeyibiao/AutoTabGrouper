# Auto Tab Grouper

> 一个优雅的 Chrome 扩展，自动将拥有相同域名的标签页归类到同一个原生的“标签组”中，帮助您保持浏览器的整洁。

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](https://opensource.org/licenses/MIT)

## 🌟 特性功能

- 🎯 **全自动分组**：无论是新开页面还是在原页面跳转，只要域名发生变动，就会实时被归入相应的标签组。
- 🎨 **智能着色与命名**：自动提取友好域名（如 `github.com`）作为组名，并基于域名 Hash 分配固定的内置颜色，确保同一网站的颜色始终一致。
- ⚡ **无缝整理**：即使是**单个**标签页，也会被自动打包成分组。
- 🧹 **边缘页后置**：对于没有域名的页面（如新标签页、设置页、`chrome://` 内部页面），扩展将自动把它们移动到所有标签栏的最末尾（最右侧/下方），绝不干扰您的主力工作区。
- 🚀 **即装即用**：安装此扩展或更新时，即刻对当前窗口已有标签页进行全局整理。

## 📦 安装说明

由于目前直接提供源码，您可以通过 Chrome 的 **开发者模式** 轻松加载此扩展。

### 手动安装（开发者模式）

1. 下载或 Clone 本仓库到本地：
   ```bash
   git clone https://github.com/your-username/AutoTabGrouper.git
   ```
2. 打开 Chrome 浏览器，在地址栏输入 `chrome://extensions/` 并回车。
3. 在页面右上角，开启 **“开发者模式” (Developer mode)**。
4. 点击左上角的 **“加载已解压的扩展程序” (Load unpacked)**。
5. 选择您刚刚下载 / Clone 的 `AutoTabGrouper` 目录。
6. 完成！扩展会立刻在后台开始为您整理当前窗口里的所有标签页。

## 🛠 技术细节

本扩展采用 Manifest V3 构建，追求极致的轻量与底层安全：
- 使用 **Background Service Worker** 进行非持久化的事件监听，只有在新建或更新了标签页时才会短暂激活，**几乎零内存占用**。
- 极简化权限申请，仅需 `tabs`（获取标签当前信息）与 `tabGroups`（操作 Chrome 原生标签组界面）两项核心权限。
